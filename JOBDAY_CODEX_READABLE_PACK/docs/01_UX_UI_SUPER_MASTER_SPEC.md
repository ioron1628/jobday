# JOBDAY SUPER MASTER PROMPT
## Brand · UX/UI · Product Architecture · Trust & Safety · Implementation Plan

당신은 JOBDAY의 수석 제품전략가, 브랜드 디렉터, UX 리서처, UI 디자인 리드,
콘텐츠 플랫폼 아키텍트, Staff Frontend Engineer, Backend Architect,
Trust & Safety 책임자 역할을 동시에 맡습니다.

이 작업의 목표는 단순히 기존 웹사이트를 예쁘게 고치는 것이 아닙니다.

JOBDAY의 사업 비전 전체를 사용자가 하나의 자연스럽고 세련된 경험으로 느끼도록
브랜드, 정보구조, 사용자 흐름, 디자인 시스템, 데이터 구조와 구현 순서를
처음부터 다시 정렬하는 것입니다.

이번 프롬프트는 큰 방향과 품질 기준을 통제하는 최상위 지침입니다.
하위 작업 프롬프트, 기존 코드, 임시 디자인보다 우선합니다.
단, 저장소에 더 구체적인 보안·법률·데이터 마이그레이션 지침이 있으면
충돌 내용을 보고하고 임의로 무시하지 마십시오.

0. 프로젝트 정체성
────────────────────────────────────────

프로젝트명:
JOBDAY

브랜드 문장:
모든 직업과 체험의 세계

핵심 정의:
실제 직업 종사자와 유경험자가 자신의 일과 삶을 이야기하고,
사람들이 그 이야기를 통해 직업을 발견하고,
질문하고, 관련 일자리를 찾고, 필요한 상품과 기업을 만나는
직업 미디어 기반 플랫폼입니다.

JOBDAY는 다음 다섯 기능을 보유하지만,
사용자에게 다섯 개의 별도 사업처럼 보여서는 안 됩니다.

1. 직업인 영상·오디오·팟캐스트 미디어
2. 직업별 경험·질문 커뮤니티
3. 작업·구인·구직 기능
4. 직업 관련 상품을 다루는 JOBDAY SHOP
5. 기업 채용광고·브랜딩·협찬·업체 광고

모든 기능은 반드시 다음 공통 객체를 중심으로 연결합니다.

- Occupation
- Person / Worker Experience
- Episode
- Community Topic
- Job Posting
- Product
- Company
- Campaign

핵심 선순환:

직업인의 이야기
→ 직업에 대한 관심과 신뢰
→ 직업 팔로우와 커뮤니티
→ 관련 일자리 탐색
→ 관련 상품·교육·업체 발견
→ 기업 채용·광고 매출
→ 더 좋은 콘텐츠와 검증 시스템에 재투자

────────────────────────────────────────
1. 절대적으로 피해야 하는 결과
────────────────────────────────────────

다음과 같은 결과를 만들면 실패입니다.

- 안전모, 공구, 검정·노랑 경고색에 의존하는 전형적인 노동 플랫폼
- 건설·생산직만을 위한 서비스처럼 보이는 디자인
- 팟캐스트, 게시판, 구인, 쇼핑, 광고가 각각 따로 노는 포털
- 첫 화면에 모든 기능과 배너를 한꺼번에 노출하는 복잡한 UI
- 관공서, 구인구직 사이트, 중고거래 앱을 섞어놓은 인상
- AI가 만든 듯한 그라데이션·유리 효과·둥근 카드의 무분별한 반복
- 텍스트 밀도가 지나치게 높고 위계가 없는 화면
- 의미 없는 통계 숫자와 가짜 후기
- 아직 작동하지 않는 미래 기능의 빈 메뉴와 placeholder
- 실제 사람이 아니라 기능 아이콘과 일러스트가 중심인 브랜드
- 자극적인 회사 폭로·비방·혐오를 조회수 수단으로 이용하는 구조
- 결제 광고가 검증과 편집권보다 우선하는 구조
- 모바일에서 데스크톱 화면을 줄여놓은 듯한 UI
- 유명 콘텐츠 채널의 로고·색상·레이아웃을 직접 복제하는 디자인

워크맨 등 대중적인 직업 콘텐츠가 가진
“누구나 가볍게 접근할 수 있는 세련됨, 인물 중심의 호기심, 명확한 콘텐츠 위계,
짧고 강한 카피, 대중성과 편집감”은 참고할 수 있습니다.

그러나 특정 브랜드의 로고, 캐릭터, 색 조합, 썸네일 문법,
화면 레이아웃을 복제하지 마십시오.

JOBDAY만의 독립적인 브랜드 언어를 만들어야 합니다.

2. 브랜드가 전달해야 하는 인상
────────────────────────────────────────

JOBDAY는 다음 성격을 동시에 가져야 합니다.

- 사람 냄새가 나지만 촌스럽지 않음
- 현실적이지만 거칠거나 위협적이지 않음
- 젊고 대중적이지만 가볍거나 유치하지 않음
- 전문적이지만 관공서나 HR 솔루션처럼 딱딱하지 않음
- 다양한 직업을 다루지만 정체성이 흐려지지 않음
- 재미있는 이야기가 있지만 폭로 채널처럼 보이지 않음
- 일을 구할 수 있지만 단순 구인 게시판처럼 보이지 않음
- 상품을 살 수 있지만 종합 쇼핑몰처럼 보이지 않음
- 기업이 광고할 수 있지만 광고판처럼 보이지 않음

────────────────────────────────────────
3. 시각 디자인 방향
────────────────────────────────────────

디자인의 중심은 “직업”이 아니라 “그 일을 살아가는 사람”입니다.

사진 원칙:

- 실제 인물의 표정, 손, 작업환경, 일상적 순간을 중심으로 사용
- 과도하게 연출된 스톡사진처럼 보이지 않게 함
- 사무직, 서비스직, 기술직, 예술직, 공공직, 자영업, 플랫폼 노동,
  야간·교대직, 외국인 근로자까지 균형 있게 표현
- 특정 성별, 나이, 국적, 육체노동 이미지에 치우치지 않음
- 익명 출연자의 경우 정체성을 보호하면서도 사람의 온기를 살리는 촬영 방식 사용

권장 시각 톤:

- Editorial media와 현대적 라이프스타일 플랫폼의 중간
- 충분한 여백
- 강한 타이포그래피 위계
- 인물 중심의 큰 썸네일
- 콘텐츠가 먼저 보이고 UI 장식은 뒤로 물러남
- 밝은 배경과 깊은 잉크 계열의 대비
- 포인트색은 신뢰와 생동감을 주되 경고색처럼 보이지 않게 사용
- 한 화면에서 핵심 강조색은 1개, 보조 강조색은 최대 1개
- 아이콘은 기능 보조용으로만 사용
- 의미 없는 이모지 사용 금지

초기 권장 색상 토큰을 제안하되,
현재 브랜드 자산을 감사한 후 최종 선택하십시오.

권장 출발점:

- Ink: #17191D
- Paper: #F7F5EF
- Surface: #FFFFFF
- Muted Surface: #F0F1ED
- Brand Green: #258A68
- Brand Green Dark: #17634B
- Editorial Accent: #E66A49
- Text Secondary: #686D73
- Border: #DBDDD8
- Success, Warning, Danger는 의미 기반 토큰으로 분리

색상은 하드코딩하지 말고 semantic design token으로 관리합니다.

서체 원칙:

- 한국어 본문 가독성을 최우선
- 제목은 개성과 밀도를 가지되 과도하게 장식적이지 않음
- 숫자, 급여, 시간, 경력 표기가 안정적으로 정렬
- 최소 본문 크기와 행간을 모바일에서 충분히 확보
- 시스템 폰트 fallback 제공
- 폰트 파일을 저장소에 무단 포함하지 않음

모션 원칙:

- 콘텐츠 재생, 화면 전환, 팔로우 완료 등 의미 있는 피드백에만 사용
- 과도한 parallax, spring, floating card 금지
- prefers-reduced-motion 존중
- 핵심 동작은 모션 없이도 이해 가능해야 함

4. 사용자에게 보이는 단순한 구조
────────────────────────────────────────

사업 내부에는 여러 모듈이 있지만,
사용자가 기억해야 할 핵심 행동은 다음 세 가지입니다.

1. 듣고 본다
2. 직업을 알아본다
3. 일자리를 찾는다

초기 모바일 하단 내비게이션 권장 구조:

- 홈
- 직업
- 일자리
- 내 정보

다음 기능은 초기 독립 하단 탭으로 두지 않습니다.

- 커뮤니티
- SHOP
- 업체 광고
- 기업 서비스

노출 방식:

- 커뮤니티는 직업 페이지와 에피소드 페이지 안에서 연결
- SHOP은 “현직자가 사용하는 장비”, “이 에피소드에 나온 도구”처럼 문맥 안에서 노출
- 기업광고는 관련 직업·에피소드·공고 문맥에서 명확한 광고표시와 함께 노출
- 기업 서비스는 별도 B2B 진입점과 대시보드 제공
- 관리자 기능은 완전히 분리된 Admin Console 제공

기능이 성장해 실제 사용성이 검증되면
데이터를 근거로 내비게이션 승격을 검토합니다.

────────────────────────────────────────
5. 핵심 사용자 집단
────────────────────────────────────────

다음 사용자를 별도로 설계하십시오.

A. 가볍게 직업 이야기를 즐기는 일반 청취자
목적:
- 재미있는 사람 이야기를 듣기
- 밤이나 이동 중 오디오로 소비
- 다음 콘텐츠를 발견

B. 직업을 탐색하는 사람
목적:
- 실제 업무와 생활을 이해
- 직업 진입 경로와 현실 확인
- 현직자의 경험과 질문 확인

C. 구직자
목적:
- 검증된 공고 확인
- 조건을 빠르게 비교
- 안전하게 지원
- 지원 상태 확인

D. 현직자·유경험자
목적:
- 경험 공유
- 익명 또는 실명 출연 신청
- 질문에 답변
- 자신의 직업을 올바르게 알림

E. 외국인 근로자
목적:
- 이해 가능한 언어로 콘텐츠와 공고 확인
- 고용조건과 지원 가능 여부 확인
- 개인정보와 체류정보 보호
- 번역 오류 신고

F. 기업·사업자
목적:
- 회사 검증
- 공고 작성
- 지원 현황 확인
- 채용 브랜딩 콘텐츠 문의
- 광고·협찬 문의

G. 편집자·운영자·검수자
목적:
- 출연자 관리
- 콘텐츠 워크플로
- 위험 발언 검수
- 신고·제재
- 기업과 공고 검증
- 광고와 협찬 승인
- 감사 로그 확인

각 사용자 집단마다 다음을 문서화하십시오.

- 첫 방문 동기
- 가장 중요한 첫 행동
- 성공 상태
- 재방문 이유
- 이탈 원인
- 민감정보
- 필요한 신뢰 신호
- 접근성 고려사항

6. JOBDAY의 핵심 정보구조
────────────────────────────────────────

플랫폼의 중심 객체는 Occupation입니다.

직업 허브 예시:

용접공
├── 직업 소개
├── 실제 종사자 에피소드
├── 업무환경과 진입경로
├── 질문과 경험담
├── 현재 일자리
├── 관련 자격·교육
├── 실제 사용 장비
└── 관련 기업·콘텐츠

모든 Episode, Job Posting, Community Topic, Product, Company Campaign은
가능하면 하나 이상의 Occupation과 연결되어야 합니다.

Occupation taxonomy 요구사항:

- 직업 대분류, 중분류, 세부 직업
- 별칭과 현장 용어
- slug
- 다국어 표시명 확장 가능
- 관련 직업
- 진입 경로 태그
- 근무환경 태그
- 업무 특성 태그
- 공개 상태
- 검수 상태
- 근거 출처와 갱신 시점

직업을 정확히 모르는 사용자도 다음 감각적 기준으로 탐색할 수 있어야 합니다.

- 사람을 상대하는 일
- 혼자 집중하는 일
- 몸을 쓰는 일
- 기술을 익히는 일
- 밤에 하는 일
- 이동이 많은 일
- 실내에서 하는 일
- 자격증 없이 시작 가능한 일
- 외국인이 지원 가능한 일
- 경력 전환이 쉬운 일
- 창의적인 일
- 안정성을 중시하는 일

8. 홈 화면 구조
────────────────────────────────────────

첫 화면에서 사업 전체를 설명하지 마십시오.
콘텐츠를 먼저 경험하게 하십시오.

권장 순서:

A. 오늘의 대표 이야기
- 큰 인물 이미지 또는 영상
- 직업명
- 강한 한 문장
- 즉시 재생
- 30~60초 미리듣기
- 익명·음성변조·협찬 여부

B. 지금 많이 듣는 직업 이야기
- 가로 스크롤
- 인물 중심 썸네일
- 직업명, 제목, 길이
- 영상·오디오 상태
- 재생 진입이 한 번의 탭으로 가능

C. 이런 일은 어떠세요?
- 직업명이 아닌 생활·성향 기반 탐색

D. 관심 직업의 새로운 이야기
- 로그인 사용자 개인화

E. 현재 올라온 일자리
- 관심 직업과 지역 기반
- 너무 많은 공고를 홈에 노출하지 않음

F. 현직자들의 질문과 경험
- 검수된 고품질 콘텐츠 우선
- 무작위 익명 잡담처럼 보이지 않게 함

G. SHOP과 광고
- 홈 핵심 영역을 차지하지 않음
- 사용자 관심과 콘텐츠 문맥이 있을 때만 제한적으로 표시

9. 에피소드 경험
────────────────────────────────────────

에피소드 페이지는 JOBDAY의 중심 전환 화면입니다.

상단:

- 영상 또는 오디오 플레이어
- 스크롤 후에도 유지되는 미니 플레이어
- 제목
- 출연자의 직업과 경력
- 공개명 또는 익명 상태
- 음성변조·모자이크 여부
- 촬영일과 게시일
- 협찬·광고 표시
- 직업 태그
- 저장, 공유, 팔로우

본문:

- 에피소드 소개
- 타임스탬프 챕터
- 대본 또는 자막
- 핵심 문장
- 사실관계 보충
- 관련 자료와 출처
- 편집상 익명화 안내
- 정정·신고 요청
- 후일담

하단 연결:

- 이 직업 알아보기
- 관련 다른 에피소드
- 현직자에게 질문
- 관련 일자리
- 실제 사용 장비
- 출연 신청

밤에 듣는 사용자를 고려하십시오.

- 화면을 끄거나 다른 화면으로 이동해도 재생 유지가 가능한 범위 검토
- 수면 타이머
- 재생속도
- 이어듣기
- 긴 대화에서 위치 기억
- 과도한 자동재생 금지
- 갑작스러운 큰 음량의 광고·효과음 금지

10. 커뮤니티 경험
────────────────────────────────────────

커뮤니티는 익명 폭로 게시판이 아니라
직업 경험과 질문이 축적되는 지식층입니다.

게시물 유형:

- 질문
- 경험담
- 입문 조언
- 근무환경
- 장비·도구
- 자격·교육
- 외국인 근로 경험
- 사실 정정
- 에피소드 후속 질문

필수 원칙:

- 특정 업체·지점·개인을 추정할 수 있는 비방 금지
- 개인정보와 고객정보 금지
- 내부 문서, 거래처 명단, 영업비밀 금지
- 증거 없는 범죄·임금체불·괴롭힘 단정 금지
- 신고 카테고리 명확화
- 수정 이력과 운영 조치 기록
- 반복 위반자 제재
- 운영자가 작성한 것처럼 보이는 가짜 활동 금지
- 건강한 초기 밀도가 없으면 전체 게시판을 공개하지 않음

좋은 답변이 축적되도록:

- 현직자 경력 표시
- 검증 범위를 세분화
- 유용한 답변 표시
- 직업별 FAQ로 승격
- 수정·정정 기록
- 질문 중복 제안

11. 구인구직 경험
────────────────────────────────────────

JOBDAY의 채용 기능은 단순 공고 목록이 아니라
직업 콘텐츠를 통해 신뢰가 형성된 뒤 움직이는 구조입니다.

공고 카드에서 바로 보여줄 정보:

- 직업
- 회사 검증 범위
- 급여와 급여단위
- 근무지역
- 근무시간
- 계약형태
- 경력 조건
- 숙소·식사·교통 제공
- 외국인 지원 가능 여부
- 가능한 언어
- 마감일
- 최근 수정일

검증 배지는 하나의 “검증됨”으로 뭉치지 마십시오.

분리 예시:

- 사업자등록 확인
- 채용 담당자 연락처 확인
- 근무지 확인
- 급여항목 확인
- 외국인 고용 관련 입력 확인
- 과거 JOBDAY 이용 이력
- 최근 중대한 제재 없음

이 배지는 JOBDAY가 근로조건과 회사 전체를 보증한다는 의미가 아님을 설명하십시오.

초기 제외:

- 운영자의 개인별 후보자 알선
- 채용 성공 수수료
- 무검증 기업의 즉시 게시
- 민감정보가 공개되는 이력서
- 기업 간 지원자 데이터 공유
- 광고비를 내면 검증을 건너뛰는 기능

12. JOBDAY SHOP 경험
────────────────────────────────────────

SHOP은 초기에 종합몰을 만들지 않습니다.

좋은 노출:

- 이 직업의 현직자들이 실제로 사용하는 장비
- 에피소드에서 언급된 제품
- 초보자가 처음 준비할 기본 구성
- 직업별 소모품과 관리용품
- 현직자 리뷰와 사용조건
- 안전·호환·규격 정보

나쁜 노출:

- 직업과 무관한 인기 상품
- 광고비순 상품 배치
- 검증 없이 “현직자 추천” 표시
- 일반 쇼핑몰과 동일한 카테고리 확장
- 가격비교만 남는 범용 소비재
- 판매자가 직접 작성한 가짜 체험기

초기에는 외부 구매 연결과 제휴를 우선 검토하고,
반복 수요와 차별성이 검증된 카테고리만 직접 판매를 검토하십시오.

상품 데이터는 Occupation, Episode, Usage Context와 연결하십시오.

13. 기업 광고와 채용 브랜딩

직업 문맥 안에서 유용한 기업 정보를 제공해야 합니다.

상품 예시:

- 검증된 채용공고
- 직업 허브 내 관련 기업 노출
- 채용 브랜딩 인터뷰
- 협찬 에피소드
- 직업 체험 콘텐츠
- 교육·도구·서비스 광고
- 기업 페이지
- 지역·직업 타깃 광고

필수 원칙:

- 광고, 협찬, 제휴를 명확히 표시
- 광고주가 부정적 사실을 은폐하도록 편집권 행사 금지
- 일반 콘텐츠와 광고 콘텐츠의 데이터 상태 분리
- 유료라고 검증 기준 완화 금지
- 광고 노출 빈도 제한
- 사용자 신고와 광고주 관계를 분리
- 광고 심사·승인·중단 기록을 Audit Log에 보관

24. 최종 품질 기준
────────────────────────────────────────

완료된 JOBDAY는 다음 질문에 모두 “예”라고 답할 수 있어야 합니다.

- 처음 본 사람이 5초 안에 무슨 서비스인지 이해하는가?
- 직업 정보보다 사람의 이야기가 먼저 느껴지는가?
- 건설·생산직 전용 플랫폼처럼 보이지 않는가?
- 젊고 세련됐지만 유치하지 않은가?
- 팟캐스트·커뮤니티·구인·쇼핑·광고가 하나의 직업 경험으로 연결되는가?
- 첫 화면에서 사용자가 무엇을 해야 할지 명확한가?
- 모바일에서 콘텐츠를 즉시 재생할 수 있는가?
- 밤에도 편하게 들을 수 있는가?
- 공고와 기업의 검증 범위가 정직하게 표시되는가?
- 외국인 사용자의 정보와 존엄이 보호되는가?
- 광고가 편집권과 검증을 침해하지 않는가?
- 운영자가 사고를 실제로 처리할 수 있는 관리자 기능이 있는가?
- 모든 핵심 상태와 오류가 설계됐는가?
- 접근성과 성능이 디자인 이후의 보완사항이 아니라 기본값인가?
- 앞으로 모듈이 늘어도 디자인과 정보구조가 무너지지 않는가?

이 기준을 충족하지 못한 상태에서
“전체 구현 완료”라고 보고하지 마십시오.
