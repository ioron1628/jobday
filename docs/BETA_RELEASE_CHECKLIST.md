# JOBDAY 베타 릴리스 체크리스트

작성일: 2026-07-06

## 1. 현재 상태

베타 오픈 가능 여부: **조건부 가능**

로컬 Release Candidate 검수는 통과했다. Vercel Preview 주소에서 `smoke`와 `e2e`를 다시 통과시키면 베타 사용자 10명에게 보낼 수 있다.

## 2. 필수 체크리스트

| 상태 | 항목 | 메모 |
| --- | --- | --- |
| 통과 | 금지 기능 추가 없음 | 결제, 채팅, 자동매칭, 수수료 없음 |

| 통과 | `pnpm run e2e` | 회원가입, 로그인, 글쓰기, 댓글, 추천, 신고, 로그아웃 |

## 3. 베타 오픈 전 마지막 순서

1. Vercel Preview를 배포한다.
2. Preview URL을 연다.
3. Preview URL 기준으로 `pnpm run smoke`를 실행한다.
4. Preview URL 기준으로 `pnpm run e2e`를 실행한다.
5. 운영자 계정 `profiles.is_admin = true`를 확인한다.
6. 관리자 계정으로 신고 처리, 숨김, 댓글 삭제, 차단을 직접 1회 확인한다.
7. 베타 사용자에게 보낼 안내문을 준비한다.

## 4. 베타 사용자에게 공개 가능한 조건

아래가 모두 통과면 베타 사용자 10명에게 보낸다.

- `pnpm run lint` 통과
- `pnpm run typecheck` 통과
- `pnpm run build` 통과
- `pnpm run smoke` 통과
- `pnpm run qa:check` 통과
- `pnpm run e2e` 통과
- Preview 주소에서 smoke/e2e 통과
- 관리자 신고 처리 수동 확인
- 책임제한 문구 확인

## 5. 베타 중 운영자가 매일 볼 것

| 항목 | 확인 |
| --- | --- |
| 신고 목록 | 허위 구인, 개인정보 노출, 거래 문제 우선 처리 |
| 새 글 | 작업조건, 연락방법, 법적 위험 문구 확인 |
| 댓글 | 욕설, 개인정보, 실명 저격 확인 |
| 차단 사용자 | 글/댓글 작성 차단 상태 확인 |
| 공지 | 베타 안내와 책임제한 문구 유지 |

## 6. 베타 후 개선 항목

- 이미지 업로드 e2e 추가
- Storage 정책 세분화
- 관리자 QA 시나리오 자동화
- 전화번호/연락방법 노출 주의 문구 강화
- 신고 처리 템플릿
- Preview 환경 자동 검수 자동화
- 강의/샵/광고 결제는 Post-Beta에서 별도 검토
